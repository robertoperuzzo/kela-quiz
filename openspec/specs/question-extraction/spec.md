## ADDED Requirements

### Requirement: Extract text from multiple document formats
The extraction pipeline SHALL read source documents in PDF, DOCX, and ODT formats from `knoledgebase/<professor>/` directories and extract their text content.

#### Scenario: Extract text from a PDF file
- **WHEN** the extraction script processes a `.pdf` file from a knowledge base folder
- **THEN** the full text content of the PDF is extracted, preserving question/answer structure

#### Scenario: Extract text from a DOCX file
- **WHEN** the extraction script processes a `.docx` file from a knowledge base folder
- **THEN** the full text content of the DOCX is extracted, preserving question/answer structure

#### Scenario: Extract text from an ODT file
- **WHEN** the extraction script processes an `.odt` file from a knowledge base folder
- **THEN** the full text content of the ODT is extracted, preserving question/answer structure

### Requirement: Structure questions via LLM using GitHub Copilot
The extraction pipeline SHALL send extracted text to a GitHub Copilot-provided LLM to identify individual questions, extract correct answers, and format the output as structured JSON.

#### Scenario: Question with all alternatives present in source
- **WHEN** the source text contains a question with multiple-choice alternatives and a marked correct answer
- **THEN** the LLM extracts the question, all alternatives, and the correct answer index into the JSON schema

#### Scenario: Question with only the correct answer present
- **WHEN** the source text contains a question with only the correct answer (no wrong alternatives)
- **THEN** the LLM generates exactly 3 plausible but incorrect alternatives in Italian, resulting in 4 total alternatives, and marks the correct answer index

#### Scenario: User selects which LLM model to use
- **WHEN** the user runs the extraction script
- **THEN** the script allows specifying which GitHub Copilot model to use (e.g., GPT-4o, Claude) via a command-line argument

### Requirement: Output one JSON file per exam
The extraction pipeline SHALL produce one JSON file per professor/exam in the `data/` directory, containing all extracted questions.

#### Scenario: JSON output structure
- **WHEN** extraction completes for a professor's documents
- **THEN** a JSON file named `<professor>.json` is written to `data/` with an array of question objects, each containing: `id` (number), `question` (string), `alternatives` (array of 4 strings), `correct` (index 0-3)

#### Scenario: Multiple source files for one professor
- **WHEN** a professor's knowledge base folder contains multiple files
- **THEN** questions from all files are combined into a single JSON file for that professor, with unique sequential IDs

### Requirement: Flag generated alternatives
The extraction pipeline SHALL mark questions where wrong alternatives were AI-generated (not from the source document) so they can be prioritized for human review.

#### Scenario: Question with generated alternatives is flagged
- **WHEN** a question's wrong alternatives were generated by the LLM (not present in source)
- **THEN** the question object in the JSON output includes `"generated": true`

#### Scenario: Question with all original alternatives
- **WHEN** all alternatives for a question were present in the source document
- **THEN** the question object in the JSON output includes `"generated": false`
